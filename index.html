What is Serverless?

Serverless

  • Care less about Server

    Less frictions about underlying infrastructure and ops-tasks

  • Pay-As-You-Go

    You pay only the execution time.

  • Automatic Scaling

    No worrying about Scale-Out and In

  • High available

    Always available as Cloud provider available ;)

  • Developer Friendly

    Developer is not stuck to a particular programming language

  • Benefits

    • That all means, we are no longer managing underlying infrastructure and its server. #NoSSH #NoOps
    • No longer provision resources because it detects and scales automatically according the environment circumstances
    • Developer Friendly because flexible in Programming Language #DUX

  • Limitations

    • For a DevOps-Engineer it increases the toolset in managing application
    • Many different tools for monitoring
    • Machine Learning is really hard because of the big libs

Template: Serverless Flask App 

npm install -g serverless # Optional
# sls works as well
npx serverless install --url https://github.com/jolo-dev/serverless-flask --name my-flask-app

cd my-flask-app && npm run setup
# answer prompt

npx serverless deploy

# Don't forget to remove the deployment
npx sls remove

Template: Zappa

# answer prompt
cd *project_slug*
python3 -m venv .env
source .env/bin/activate

When choosing Django

Add this to setting.py 

# In {{ project_slug }}/settings.py
# You need to add the URL of AWS to ALLOWED_HOSTS
# In this example it looks like this
ALLOWED_HOSTS = ["p0m8jb2g5b.execute-api.eu-central-1.amazonaws.com"]

Update your Project 

zappa update dev
zappa manage dev "collectstatic --noinput" # Add your static files to your S3-Buckets
# Remove your work
zappa undeploy dev
# Note: You need to remove your S3 Buckets manually or by
chmod +x remove_buckets.sh

  • Serverless Framework

    • + Multi-Vendor
    • + Easy and Fast to setup
    • + Many Plugins
    • + Cloudformation in YAML
    • + Support of Many Programming Languages
    • - Written in NodeJS and not Python-Only
    • - Many Configurations to make it Python-Ready

  • Zappa
    Framework

    • +/- Python Only
    • + Super fast
    • + Easy Migration of WSGI Applications
    • +/- AWS Only
    • - Documentation is overwhelming (personally, I don't like it)
    • - No Plugins
    • - Manual configuration for YAML
    • - After removal, no remove of S3-Bucket

Conclusion

  • New Trend

    There is a new shift towards Serverless Architecture

  • Ecosystem

    Many (open-source) solutions for deploying Serverless application

  • Servers are still there

    There is still some overhead in terms of configuration. But #NoSSH

  • DevOps Culture

    It embraces the DevOps Culture. Ops and Devs are coming closer.

  • Depends on the Use Case

    Not everything is solveable via a single function. Or it might be that you want to manage your servers/infrastructure.
